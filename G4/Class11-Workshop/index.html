    5. Create a total field at the bottom that will show the total cost of all the items added in the list
    6. Validate the input so that the user can't add ingredient names less than 2 characters or price/quantity below 0
    7. Show an error message in the HTML to the user if he has invalid input
    8. BONUS: Create the checkout element above only after you have finished the form and the list
    9. The user needs to have total cash and then when he clicks on the pay order button the total from the list is removed from the cash and the order is reset
    10. If the user makes an order that goes above his available cash then show him an error in the html telling him he has insufficent funds
    11. With this the user is going to be able to make multiple orders until he runs out of money
