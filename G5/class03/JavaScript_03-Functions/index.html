Switch statements

  • Starts with a variable called switch value
  • Each case indicates a possible value for that variable
  • Have a default option if no match is found

Structure


	switch(someVal) {
		case 'value1':
			//do something;
			break;
		case 'value2':
			//do something else;
			break;
		default:
			//no value match found, do default option
			break; //this is optional
	}

"fall through" scenario

  • If you omit break from a case, and that case matches or runs, execution will continue with the next case's statements regardless of that case matching


	switch(someVal) {
		case 'value1':
		case 'value2':
			//do something;
			break;
		case 'value3':
			//do something else;
			break;
		default:
			//no value match found, do default option
			break; 
	}

What is a function?

  • Break up the code's tasks into reusable pieces
  • Generally a named section of code that can be "called" by name, and the code inside it will be run each time

Function declaration

  • Put the keyword function, give it a name and then write the statements needed to achieve its task inside curly braces (yes, a code block indeed)

function(){
	//... some code that does something
}

Function execution

  • Or calling the function
  • Execute all statements between function's curly braces with just one line of code
  • ...as many time as you want...


							function myFunction() {
	//loops, ifs, statements, anything;
	//...
}



myFunction();
myFunction();
myFunction();

  • Code in function won't run, unless explicitelly called
  • Doesn't matter where are defined
    • Best practice: define your functions before you call them

Scope

  • Scope determines the accessibility (visibility) of these variables.
  • Variables defined inside a function are not accessible (visible) from outside the function.
  • A collection of variables as well as the rules for how those variables are accessed by name.
  • JavaScript has function scope: Each function creates a new scope.
  • No matter where a function is invoked from, or even how it is invoked, its scope is only defined by where the function was declared

Variable scope

  • If you try to access a variable's value in a scope where it's not available, you'll get a ReferenceError thrown
  • If you try to set a variable that hasn't been declared, you'll end up creating a variable in the top-level global scope (bad!)

Student Exercise - #2

  • Decription: Celsius to Fahrenheit formula: X°C x 1.8 + 32
  • Action: Write a function that will be able to convert Celsius to Fahrenheit and display the result
  • Bonus: Write another function that will be able to convert Fahrenheit to Celsius and display the result. Formula: (5/9) * (Fahrenheit-32);
  • Time to accomplish: 15 minutes.

Exercise 2 Solution


//Declaration of both functions
function farToCel(celsius){
	let result = celsius * 1.8 + 32;
	return result;
}
function celToFar(farenheits){
	let result = (5/9) * (farenheits-32);
	return result;  
}

//The person chooses what to calculate
let personChoice = prompt("Enter F for farenheit and C for celsius");
let personInput = parseInt(prompt("Please enter value:"));
if(personChoice === "F"){
	console.log(`${celToFar(personInput)}C`);
} else if(personChoice === "C"){
	console.log(`${farToCel(personInput)}F`);
} else {
	console.log("ERROR!");
}
