VARIABLES

  • Variables names must begin with a letter or underscore
    • ex. letters, numbers, _, $ (cannot start with number)
  • Variable names are case-sensitive
    • ex. firstName , start_date , workDays...
  • Variables are untyped (VALUE IS THE TYPE!)
  • Variables declared within a function are LOCAL to that function (accessible only within that function)
  • Variables declared outside a function are GLOBAL (accessible from anywhere on the page)
  • Variables should be named in english language

Undefined vs Null

In JavaScript, UNDEFINED means a variable has been declared but has not yet been assigned a value, such as:

var TestVar;
alert(TestVar); //shows undefined
alert(typeof TestVar); //shows undefined

null is an assignment value. It can be assigned to a variable as a representation of no value:

var TestVar = null;
alert(TestVar); //shows null
alert(typeof TestVar); //shows object

From the preceding examples, it is clear that undefined and null are two distinct types: undefined is a type itself (undefined) while null is an object.

null === undefined // false
null == undefined // true
null === null // true
